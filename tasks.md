# План задач: веб-панель поставщика (Supplier Portal)

План выполнения требований из `requirements.md`. Задачи сгруппированы по фазам; внутри фазы порядок выполнения указан явно или через зависимости.

**Архитектура: Feature-Sliced Design (FSD).** Слои (снизу вверх): `shared` → `entities` → `features` → `widgets` → `pages` → `app`. Импорты только в сторону нижележащих слоёв. В каждом слайсе — сегменты `ui`, `api`, `model`, `lib`, `config` по необходимости. Публичный API слайса — через `index.ts`.

## Зависимости от бэкенда (надо уточнить перед/во время разработки)

- Точные пути auth: login, refresh, logout.
- Наличие и путь GET /api/categories (или способ получить список категорий).
- Наличие и путь GET складов для поставщика (если нужен выбор складов в форме продукта).
- Наличие GET /api/notifications и PATCH/PUT для пометки прочитанным.
- Наличие endpoint смены пароля и обновления профиля (PUT /api/profile и т.п.).
